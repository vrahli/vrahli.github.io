Student Projects


Here's a non-exhaustive list of projects I would be interested in supervising. Click the titles below to expand the sections.

I'm interested in projects related to implementing and/or formalizing programming languages. For example, distributed systems are notoriously hard to implement and get right. Projects related to this could be:

  • designing domain specific languages (DSL) to help implement distributed systems can help ease this task
  • a related project could be the formalization of such DSLs
As another example, the Nuprl theorem prover is both a programming language and a framework to formally prove mathematical results. It is however lacking, among other things, a modern interface and a compiler. Projects related to this, could be to
  • implement a modern interface for Nuprl
  • implement an interpreter for Nuprl
  • implement a compiler for Nuprl
  • implement extensions of Nuprl, for example with side effects
We have formalized Nuprl in Coq, which is another theorem prover, and have verified some of its properties. Related projects can be for exampple:
  • extend Nuprl and its formalization with new features
  • formalize other kinds of programming languages



I'm interested in projects related to formalizing of logical theories. For example, As part of our formalization of Nuprl in Coq, we have formalized Nuprl's type theory in Coq; and as part of our formalization of KeYmaera in Coq, we have formalized its core in Coq. I would, among other things, be interested in extending these theories with new features, or to formalize other kinds of logical systems.



Fault-tolerant distributed systems such as PBFT are nowadays heavily used in blockchain infrastructures (see our Asphalion paper for pointers). There are many proof-of-concepts that have been discussed in the literature, but that have never been fully implemented, and that are typically lacking important features such as state transfer or group membership changes. I'm interested in exploring such systems and extending them with features such as the ones discussed above. For example, one could implement

  • MinBFT
  • Pistis



In addition to implementing these distributed system, I'm interested in verifying their correctness, for example using our Asphalion tool.



In order to implement distributed systems, one typically needs to implement some message passing infrastructure. I'm interested in building modular, configurable, and lightweight message passing infrastructures that make it easy to deploy distributed systems.



Byzantine fault-tolerant systems are more and more used in blockchain systems. However, they typically need to be adapted to fit the needs of those systems. I'm interested in supervising projects that aim at designing such tailored solutions.
